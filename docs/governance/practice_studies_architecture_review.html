Governance UX endpoint — architectural significance
Innovation
Runtime governance state = đột phá cho AI agent systems
Hầu hết AI governance là static markdown (README, docs). ACP deliver governance state via API — operator có thể curl /governance/status và biết ngay milestones, 6-layer map, verify commands, CS-01..06, PB gate trong 1 JSON call. Đây là self-describing governance — rất phù hợp với autonomous agent context.
Design đúng
/health vs /governance/status phân tách đúng concern
/health = liveness/readiness signal (k8s, CI, quick check). /governance/status = human-readable governance checklist. Phân tách này tránh bloating /health với governance noise. Pattern này nên được documented trong CONTRIBUTING.md như design principle cho forks.
SSOT đúng chỗ
governance_catalog.py là SSOT — không phải markdown
CS-01..06 được serve từ core/governance_catalog.py — không phải hardcode trong endpoint, không phải chỉ markdown. Điều này đảm bảo governance catalog có thể được unit-tested và versioned cùng với code changes. Đây là điểm đúng nhất về kiến trúc trong toàn bộ practice evidence.
